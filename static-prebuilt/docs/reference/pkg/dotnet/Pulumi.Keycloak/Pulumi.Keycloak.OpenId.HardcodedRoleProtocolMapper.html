Class HardcodedRoleProtocolMapper

# keycloak.openid.HardcodedRoleProtocolMapper

Allows for creating and managing hardcoded role protocol mappers within Keycloak.

Hardcoded role protocol mappers allow you to specify a single role to always map to an access token for a client. Protocol mappers can be defined for a single client, or they can be defined for a client scope which can be shared between multiple different clients.

Example Usage (Client)

using Pulumi;
using Keycloak = Pulumi.Keycloak;

class MyStack : Stack
{
public MyStack()
{
    var realm = new Keycloak.Realm(&quot;realm&quot;, new Keycloak.RealmArgs
    {
        Enabled = true,
        Realm = &quot;my-realm&quot;,
    });
    var role = new Keycloak.Role(&quot;role&quot;, new Keycloak.RoleArgs
    {
        RealmId = realm.Id,
    });
    var openidClient = new Keycloak.OpenId.Client(&quot;openidClient&quot;, new Keycloak.OpenId.ClientArgs
    {
        AccessType = &quot;CONFIDENTIAL&quot;,
        ClientId = &quot;test-client&quot;,
        Enabled = true,
        RealmId = realm.Id,
        ValidRedirectUris = 
        {
            &quot;http://localhost:8080/openid-callback&quot;,
        },
    });
    var hardcodedRoleMapper = new Keycloak.OpenId.HardcodedRoleProtocolMapper(&quot;hardcodedRoleMapper&quot;, new Keycloak.OpenId.HardcodedRoleProtocolMapperArgs
    {
        ClientId = openidClient.Id,
        RealmId = realm.Id,
        RoleId = role.Id,
    });
}

}

Example Usage (Client Scope)

using Pulumi;
using Keycloak = Pulumi.Keycloak;

class MyStack : Stack
{
public MyStack()
{
    var realm = new Keycloak.Realm(&quot;realm&quot;, new Keycloak.RealmArgs
    {
        Enabled = true,
        Realm = &quot;my-realm&quot;,
    });
    var role = new Keycloak.Role(&quot;role&quot;, new Keycloak.RoleArgs
    {
        RealmId = realm.Id,
    });
    var clientScope = new Keycloak.OpenId.ClientScope(&quot;clientScope&quot;, new Keycloak.OpenId.ClientScopeArgs
    {
        RealmId = realm.Id,
    });
    var hardcodedRoleMapper = new Keycloak.OpenId.HardcodedRoleProtocolMapper(&quot;hardcodedRoleMapper&quot;, new Keycloak.OpenId.HardcodedRoleProtocolMapperArgs
    {
        ClientScopeId = clientScope.Id,
        RealmId = realm.Id,
        RoleId = role.Id,
    });
}

}

Argument Reference

The following arguments are supported:

  • realm_id - (Required) The realm this protocol mapper exists within.
  • client_id - (Required if client_scope_id is not specified) The client this protocol mapper is attached to.
  • client_scope_id - (Required if client_id is not specified) The client scope this protocol mapper is attached to.
  • name - (Required) The display name of this protocol mapper in the GUI.
  • role_id - (Required) The ID of the role to map to an access token.
